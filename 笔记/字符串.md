### AC自动机 (AC-Automaton)

- 由于KMP只能解决单个模式串匹配的问题，而如果对于一个匹配串，有多个模式串，问在匹配串中出现多少次？
  - 朴素的 KMP 要对于每个模式串都跑一次
  - 所以我们考虑把所有模式串合并成一个 TRIE 树，从根到每个节点都代表一个前缀，与KMP类似的，我们需要找到它的失配指针（fail指针）。与KMP不同的是，我们要找的指针是指向对于 **所有** 前缀中最长的可以匹配的当前后缀节点，而KMP是保证在自己串内的。

#### 实现

- 使用 BFS 遍历 TRIE
- 对于每个节点的所有儿子，都与 KMP 一样不停的跳 fail 指针，并检查是否可以接上，放入队列
- **fail的儿子是儿子的fail**

### 后缀数组(Suffix Array)

- 对于一个字符串的所有后缀进行排序后的数组：

  - 例如 "abacd" => {"abacd", "acd", "bacd", "cd", "d"}
  - 对应的下标的即 SA[] = {1, 3, 2, 4, 5}
  
- 如何高效求 SA ？

  - 基数排序 + 倍增 $O(n \log n)$ 

  - 分治排序 DC3 $O(N)$

  - 诱导排序 SA-IS $O(N)$ [诱导排序与SA-IS算法 - riteme.site](https://riteme.site/blog/2016-6-19/sais.html)

#### 基数排序 + 倍增 实现

![image-20260427185704625](C:\Users\XK\AppData\Roaming\Typora\typora-user-images\image-20260427185704625.png)

因为字典序存在后效性，前面已经大的序列不可能会又比前面小，所以可以使用双关键字排序，前 $i$ 位的排名本质上可以由 $\lfloor i / 2 \rfloor$ 的顺序倍增得到，因为前半段字典序和后半段字典序大小本质在上一次倍增的过程中就已经比较完了，于是可以开一个大小刚好是 $O(N)$ 的桶用于排序，然后每一次倍增的更新桶。

而这里还有一个问题，如果倍增超过了序列长度怎么办？不难想到其实不会影响结果，因为对于一个串来说，后面再添加字符只会让答案更劣，所以如果原本的序列已经可以确定答案，则倍增超过了不会影响本质结果，所以尽管倍增超过了 $|S|$ ，实际上不会影响。

而且当我们排序到 rank 中**没有**相同数字的时候，实际上就可以终止倍增过程了，**因为 rank 在之后的过程中不会发生任何改变了**。

处理桶是 $O(N)$ 的，次数则是桶的 $\log$ 次，于是结果就是 $O(N\log N)$ .

```cpp
struct SA { // 0-based
	int m = 128;
	string s;
	int n;
	v<I> rk, oldrk, sa, id, cnt;
	SA(const string &s) : s(" " + s), n(sz(s)) {
		rk.resize(2 * n + 10);
		sa.resize(n + 10);
		id.resize(n + 10);
		cnt.resize(max(m, n) + 10);
		oldrk.resize(2 * n + 10);
		build();
	}

	void build() {
		int p = 0;
		for (int i = 1; i <= n; i++) cnt[rk[i] = s[i]]++;
 	 	for (int i = 1; i <= m; i++) cnt[i] += cnt[i - 1];
  		for (int i = n; i >= 1; i--) sa[cnt[rk[i]]--] = i;

        for (int w = 1;; w <<= 1, m = p) {  // m = p 即为值域优化
            int cur = 0;
            for (int i = n - w + 1; i <= n; i++) id[++cur] = i;
            for (int i = 1; i <= n; i++)
              if (sa[i] > w) id[++cur] = sa[i] - w;
            fill(all(cnt), 0);
            for (int i = 1; i <= n; i++) cnt[rk[i]]++;
            for (int i = 1; i <= m; i++) cnt[i] += cnt[i - 1];
            for (int i = n; i >= 1; i--) sa[cnt[rk[id[i]]]--] = id[i];
            p = 0;
            oldrk = rk; 
            for (int i = 1; i <= n; i++) {
              if (oldrk[sa[i]] == oldrk[sa[i - 1]] &&
                  oldrk[sa[i] + w] == oldrk[sa[i - 1] + w])
                rk[sa[i]] = p;
              else
                rk[sa[i]] = ++p;
            }
            if (p == n) break;  // p = n 时无需再排序
        }
	}
	v<I> getsa() {
		v<I> ans;
		rep(i, 1, sz(s)) ans.push_back(sa[i] --);
		return ans;	
	}
};

//"aabaaaab"
// aaaab - aaab - aab - aabaaaab - ab - abaaaab - b - baaaab
// 4 	   5      6     1          7    2         8   3
// 4 5 6 1 7 2 8 3
```

#### 例题

- **(LCS- Longest Common Substring)**
  - 给定两个字符串 $S, T$ ，求其最长公共子串的长度。 $(|S|, |T| \in (1, 250000])$

这里引入一个新的数组 $height$， $height[i] =  LCP(SA[i - 1], SA[i])$。

因为有 $height[rk[i]] - 1 <= height[rk[i + 1]]$ ，所以我们可以 $O(N)$ 的构造 height数组。

```cpp
	v<I> gethei() { // hei[i] = LCP(sa[i - 1], sa[i])
		v<I> hei(n + 10, 0);
		I p = 0;
		rep(i, 1, n + 1) {
			if(p) p --;
			while(s[i + p] == s[sa[rk[i] - 1] + p]) p ++;
			hei[rk[i]] = p;
		}
		v<I> ans;
		rep(i, 1, sz(s)) ans.push_back(hei[i]);
        // 注意 ans[0] 无意义，hei[i] = LCP(SA[i - 1], SA[i])
		return ans;
	}
```

- **2017 ICPC 沈阳 G. Infinite Fraction Path**
  -  给定一个字符集为 $'0' - '9'$ 的字符串 $S$ ，链接 $S[i]$ 与 $S[(i^2 + 1) \% |S|]$ 。从任意一个 $S[i]$ 出发，经过之前链接的唯一的路径，可以走出一个无限长的字符串，求走出的字符串中最大的字符串的前 $|S|$ 位。
  - $0 \lt |S| \lt 150000$

可以利用后缀数组倍增法的思想，依次算出以每个点为起点，长度是 2 的幂的字符串长度的排序，这样就可以直接用排序算出 $|S|$ 时候的字符串排序了， **关键在于每个点仅一个出度！**，复杂度显然是 $|S|\log|S|$。

### 后缀自动机 (Suffix Automaton)

- 希望访问一个字符串的所有子串
- 如果我们把所有后缀建立一个 AC 自动机的话，显然复杂度不够，所以我们考虑用一个更优的结构，便是后缀自动机
- SAM 是由单个字符串构造的 DAG (有向无环图)，从根节点开始走到任意一条路径都是 $S$ 的一个子串，那么某一个子串也就对应了一条从根节点开始的路径

- SAM : 这是一个简化了之后的对于每一个后缀建立的 TRIE

  - 每个节点是一个 endpos，我们将一系列子串放入同一个集合（所代表的路径），当且仅当终止位置一致
  - 对于一个节点，能到它的路径上的多个串互为彼此的后缀，且满足长度递减；那么我们的后缀链接，对于每个节点，指向 endpos 集合的子串的（最短子串 - 1） 的节点

  ```cpp
  struct SAM { // 0 - based
  	v<I> pa, len, siz, isend;
  	v<array<int, 26>> ch;
  	I idx = 0, pre = 0;
  	SAM(string s) {
  		ch.resize(2 * sz(s) + 10);
  		pa.resize(2 * sz(s)  + 10);
  		len.resize(2 * sz(s)  + 10);
  		siz.resize(2 * sz(s)  + 10);
  		len[0] = 0, pa[0] = -1, idx = 0, pre = 0;
  		for(auto c : s) ins(c - 'a');
  	}
  
  	void ins(int c) {
  		I p = pre, np = ++ idx;
  		pre = np; siz[np] = 1;
  		len[np] = len[p] + 1;
  		while(p != -1 && ch[p][c] == 0) ch[p][c] = np, p = pa[p];
  
  		if(p == -1) pa[np] = 0;
  		else {
  			I q = ch[p][c];
  			if(len[q] == len[p] + 1) pa[np] = q;
  			else {
  				I nq = ++ idx;
  				len[nq] = len[p] + 1;
  				ch[nq] = ch[q];
  				pa[nq] = pa[q];
  				pa[q] = pa[np] = nq;
  				while(p != -1 && ch[p][c] == q) ch[p][c] = nq, p = pa[p];
  			}
  		}
  	}
  
  	v<v<I>> getpa() {
  		v<v<I>> tree(SAM::idx + 1);
  		rep(i, 1, SAM::idx + 1) tree[pa[i]].push_back(i);
  		return tree;
  	}	
  };
  ```

  

  

  #### 例题

  可以证明，SAM 构造出的节点与回文串有个性质：

  - 每个节点表示的点内最多只有一个回文串，且该回文串一定是以当前节点为终点的最长的回文串，且是该回文串第一次出现
    - 可得：SAM 内的一个节点至多包含一个回文串，并且必定是以该点为终点的最长子串。
    - 所以你可以用 manacher 求出来每个点作为终点的最长回文子串长度，然后构造 SAM， 每次插入后检查以当前节点为结尾的回文串最大长度是否在 last 节点的表示范围内，如果是的话就记录。
